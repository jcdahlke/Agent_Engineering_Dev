How It Works

01

CLI Launch

python runner.py --topic "AI" --mode flow initializes ResearchFlow

02

@start fires

Flow maps inputs={topic, depth} onto typed FlowState, calls kickoff_research()

03

@listen launches crew

run_research_crew() builds and starts the hierarchical ResearchCrew

04

Manager delegates

Auto-Manager (gpt-4o) reads all task descriptions and dynamically assigns work to agents

05

Researcher gathers

Senior Researcher runs SerperDevTool, arXiv search, and scrapes key pages → ResearchGathering

06

Analyst synthesizes

Data Analyst receives research via context=[research_task] → outputs AnalysisResult

07

Writer drafts report

Technical Writer uses both upstream contexts → produces structured ResearchReport

08

@router quality gate

Editor reviews → EditorialReview. @router routes to publish (≥0.7) or revise

CrewAI — Strengths

  • ✓
    Minimal boilerplate — role + goal + backstory define a fully functional agent in ~10 lines
  • ✓
    Hierarchical process — auto-manager LLM dynamically delegates and re-assigns tasks; no manual routing logic
  • ✓
    Built-in memory — short-term + long-term (SQLite + embeddings) + entity memory with a single memory=True flag
  • ✓
    Task context chaining — context=[task1, task2] automatically injects upstream outputs into task prompts
  • ✓
    Pydantic structured outputs — output_pydantic=Model on any task enforces typed, validated responses
  • ✓
    Flows for production — @start/@listen/@router provides a full state machine above crews
  • ✓
    Built-in rate limiting — max_rpm on the Crew throttles all LLM calls collectively; no custom retry logic needed
  • ✓
    Rich callback system — task_callback + step_callback give per-task and per-step visibility
  • ✓
    Extensive tool ecosystem — 20+ built-in crewai-tools (search, scraping, DB, GitHub, email) plus easy custom @tool functions
  • ✓
    Framework-opinionated — convention over configuration reduces decision fatigue; sensible defaults for most use cases

CrewAI — Limitations

  • ✗
    Opaque hierarchical control — the manager's delegation logic is a black box; hard to debug unexpected task ordering
  • ✗
    No explicit graph topology — unlike LangGraph, there's no visual or code-level graph; agent connections are implicit and dynamic
  • ✗
    Memory adds latency and cost — memory=True triggers OpenAI embedding API calls for every task output stored and recalled
  • ✗
    Manager token overhead — Process.hierarchical generates extra LLM calls per delegation round; can 2-3× token usage vs. sequential
  • ✗
    No native streaming — no token-level or node-level streaming comparable to LangGraph's stream_mode; output appears only after task completion
  • ✗
    No built-in HITL interrupt/resume — no equivalent of LangGraph's interrupt() + Command(resume=...) pattern for human-in-the-loop
  • ✗
    Context window explosion — all context=[...] task outputs are injected verbatim into the next task's prompt; can balloon token usage on long reports
  • ✗
    Version coupling — crewai and crewai-tools versions must be compatible; mismatches cause silent failures or import errors
  • ✗
    @router is single-level — Flows support one routing layer; nested conditional graphs require multiple Flow classes with explicit wiring
  • ✗
    SERPER_API_KEY dependency — SerperDevTool raises ValueError at instantiation if the env var is missing; requires graceful guards in production code

When to Use CrewAI

⚡

Rapid Research Automation

Get a working multi-agent research pipeline in under 50 lines. Ideal when speed to prototype matters more than flow control.

👥

Role-Based Agent Teams

When your problem naturally maps to human team roles — researcher, analyst, writer, editor — CrewAI's role metaphor is the most intuitive fit.

📄

Document Generation Pipelines

Reports, summaries, audits, and documentation driven by specialist agents. Pydantic structured outputs ensure consistent formatting.

📊

Competitive Intelligence

Researcher + analyst + writer crews for automated market analysis, product teardowns, and industry trend reports.

🏭

Content Production at Scale

Blog posts, product descriptions, SEO content, and documentation generated by coordinated agent teams with quality review gates.

🚀

Prototype-to-Production

Start with Process.sequential for prototyping, add hierarchical for complexity, wrap in Flows for production deployment.

Choose CrewAI when you want fast, role-based agent teams with minimal boilerplate and built-in memory.
Choose LangGraph when you need explicit graph topology, human-in-the-loop interrupts, or token-level streaming.
