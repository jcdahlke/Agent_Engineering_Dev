What is OpenAI Agent SDK?

🧠

Agent

First-class agent objects with name, instructions, tools, handoffs, model, model_settings, hooks, and input_guardrails — all in one dataclass.

🔀

Native Handoffs

Agents transfer control to each other via handoff(). The full conversation history is preserved across every transfer — no manual context passing.

🔧

Function Tools

@function_tool auto-generates a JSON schema from Python type annotations. A RunContextWrapper[T] first param is injected automatically for shared state access.

🛡️

Guardrails

@input_guardrail / @output_guardrail run lightweight validation agents before or after the main agent. Tripping the wire aborts the run immediately.

📡

Built-in Tracing

trace() context manager wraps any run and sends structured telemetry to OpenAI's tracing dashboard — zero config, no external service needed.

⚡

Streaming

Runner.run_streamed() returns a RunResultStreaming. Iterate stream_events() for raw tokens, handoff events, and completed run items in real time.

AgentHooks lifecycle callbacks

class ResearchHooks(AgentHooks[ResearchContext]):
    async def on_start(self, ctx, agent):
        console.print(f"▶ {agent.name}")

    async def on_tool_start(self, ctx, agent, tool):
        console.print(f"  → {tool.name}")

    async def on_handoff(self, ctx, agent, source):
        console.print(
            f"{source.name} → {agent.name}"
        )

How a Run Works

01

CLI Input

python runner.py --topic "X" --mode stream
Parsed by argparse; asyncio.run() starts the pipeline.

02

Guardrail Check

TopicValidator agent evaluates the topic. Unsafe or nonsensical topics trip the wire before any research begins.

03

Supervisor Starts

Runner.run(supervisor, prompt) — hooks fire on_start. Supervisor immediately hands off to Researcher.

04

Research Phase

Researcher calls web_search, arxiv_search, fetch_webpage. All findings saved via save_research_notes.

05

Analysis Phase

Analyzer loads all notes, synthesizes key findings, and saves structured AnalysisResult to shared context.

06

Writing Phase

Writer reads analysis, composes a full Markdown report with sections, then saves draft and hands off to Critic.

07

Critique & Revision

Critic scores the draft (0–10). If score < 7, it hands back to Writer with specific instructions — up to 2 revisions.

08

Final Output

save_final_report marks completion. Rich console prints the report, quality score, sources, and agent path.

Pros of OpenAI Agent SDK

  • ✅
    Minimal boilerplate — An agent is a single Agent() call. No graph builders, no decorators, no YAML configs.
  • ✅
    Native handoffs — First-class handoff() primitive carries full conversation history across agent transfers automatically.
  • ✅
    Built-in tracing — trace() sends telemetry to OpenAI's dashboard. No external observability stack required.
  • ✅
    Typed shared context — RunContextWrapper[T] gives tools type-safe access to a shared dataclass — no globals, no external DB for simple pipelines.
  • ✅
    Input & output guardrails — Safety validation is a first-class SDK primitive, not an afterthought.
  • ✅
    Structured outputs — output_type=PydanticModel forces valid JSON from any agent; extracted type-safely with final_output_as().
  • ✅
    Streaming + hooks — Real-time token streaming and full lifecycle callbacks (AgentHooks) without any custom wiring.
  • ✅
    OpenAI-native — Maintained by OpenAI; guaranteed compatibility with all new models and API features the moment they ship.

Cons of OpenAI Agent SDK

  • ⚠️
    OpenAI-only — The SDK is tightly coupled to OpenAI's API. Switching to Anthropic, Gemini, or open-source models requires a full rewrite.
  • ⚠️
    No graph visualization — Unlike LangGraph's ASCII/Mermaid output or AutoGen's topology diagrams, there's no built-in way to visualize agent connections.
  • ⚠️
    No persistent memory — The SDK has no built-in long-term memory store. You must add your own vector DB or key-value store for cross-session recall.
  • ⚠️
    Flat state model — Context is a single shared object; there's no reducer pattern, branching, or merging like LangGraph's Annotated state.
  • ⚠️
    Newer ecosystem — Released in 2025; fewer community tutorials, third-party integrations, and battle-tested patterns compared to LangChain or CrewAI.
  • ⚠️
    No native HITL — Human-in-the-loop interrupts require custom tooling. LangGraph's interrupt() + Command(resume=...) is more ergonomic.
  • ⚠️
    Limited orchestration primitives — No built-in parallel execution, retry policies on nodes, or conditional edge routing beyond what the LLM decides via handoffs.

Research Automation

Automated literature reviews, competitive analysis, and due-diligence reports across web and academic sources.

Customer Support

Triage agent routes tickets to specialist agents (billing, tech, returns). Native handoffs preserve full chat context.

Any task requiring sequential specialists: data collection → processing → validation → reporting.

Document Generation

Gather sources → outline → draft → edit → approve. Built-in guardrails catch harmful content before publishing.
