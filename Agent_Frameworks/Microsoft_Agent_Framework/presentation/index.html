Overview

What is Microsoft Agent Framework?

A lightweight, extensible Python & .NET framework from Microsoft for building multi-agent systems with flexible orchestration patterns — the convergence of AutoGen and Semantic Kernel.

🔀

5 Built-in Patterns

Sequential, Concurrent, Handoff, Group Chat, and Magentic One — all first-class citizens, no custom wiring.

🔧

Simple API

client.as_agent() creates a full agent in one call. No graph nodes, edges, or state reducers to configure.

🛡️

Middleware Pipeline

Pluggable AgentMiddleware for termination, guardrails, logging, and rate limiting — applied per-agent.

👤

Tool-Level HITL

approval_mode="always_require" on any tool gates it behind human confirmation. No graph pausing needed.

🌊

Native Streaming

Pass stream=True to any agent.run() call for live token-by-token output across the entire agent loop.

⚡

Async-First

Built on Python asyncio. All agents run concurrently by default — compose patterns with asyncio.gather().

How It Works

Agent Roles

🎯 Orchestrator
gpt-4o · temperature 0 · MaxIterationsMiddleware
  • Hub of the handoff pattern — always retains control
  • Handoff tools directly await each specialist agent
  • Reads shared research_state dict via closure
  • Middleware terminates run after N iterations
🔍 Researcher
gpt-4o-mini · internal ReAct loop (framework-managed)
  • Tavily / DuckDuckGo web search
  • arXiv academic paper search
  • Wikipedia background lookup
  • Full page scraping via BeautifulSoup
📊 Analyzer
gpt-4o · structured output via prompt engineering
  • Extracts key findings and data points
  • Outputs KEY FINDINGS, CONFIDENCE, GAPS
  • Uses extract_key_facts tool for structure
  • Signals if more research coverage is needed
✍️ Writer
gpt-4o · publish_report HITL gate
  • Synthesizes findings into structured Markdown report
  • Addresses critique from previous review cycles
  • Calls publish_report — triggers human approval gate
🔍 Critic
gpt-4o-mini · pure LLM reasoning (no tools)
  • Scores draft 0.0–1.0 with explicit rubric
  • Outputs SCORE: X.XX and DECISION: APPROVED/REVISION NEEDED
  • Orchestrator regex-parses these signals
  • Approves (≥0.7) → Orchestrator calls finish_research

Patterns

All 5 Orchestration Patterns

Agent Framework provides five built-in patterns. This demo uses Handoff — highlighted below.

🔗 Sequential

Agents run in a fixed pipeline A → B → C → D. Best for linear workflows with clear hand-off points. Simplest pattern — minimal configuration.

⚡ Concurrent

Multiple agents process input in parallel via asyncio.gather(). Best when subtasks are independent and total latency matters more than order.

🔀 Handoff — Used in This Demo

Agents hand control to each other via tool calls. Hub-and-spoke variant: Orchestrator always retains control. Best for dynamic, condition-based routing.

💬 Group Chat

Agents share a conversation; a moderator picks the next speaker (round-robin or LLM-selected). Best for collaborative brainstorming and multi-perspective analysis.

🧠 Magentic One

A Planner agent coordinates specialists with a shared scratchpad and task ledger. The Planner decomposes tasks, assigns subtasks, and synthesizes results. Best for complex, long-horizon autonomous tasks requiring multi-step planning. Adapted from Microsoft Research's Magentic-One paper (2024).

Evaluation

Pros & Cons of Microsoft Agent Framework

✓ Strengths

  • Simple API — client.as_agent() creates a working agent in one call; far less boilerplate than LangGraph
  • 5 built-in patterns — Sequential, Concurrent, Handoff, Group Chat, Magentic One without custom wiring
  • Native streaming — stream=True on any agent.run() call, including through tool loops
  • Extensible middleware — any Python class implementing AgentMiddleware plugs in instantly
  • Tool-level HITL — granular, per-tool approval gates; no graph-level pause/resume complexity
  • Async-first — built on asyncio; concurrent patterns compose naturally
  • Multi-provider — OpenAI, Azure OpenAI, Anthropic, Ollama, and more via provider packages
  • Lightweight — fewer abstractions than LangGraph/CrewAI; the framework stays out of your way

✗ Limitations

  • Early-stage — v1.0 released 2026; API may change; fewer community examples than LangGraph/CrewAI
  • No built-in persistence — unlike LangGraph's MemorySaver, state is lost if the process exits
  • No time-travel debugging — can't inspect or rewind to previous checkpoints
  • No visualization — no built-in Mermaid or ASCII graph generation
  • Manual state management — plain dict instead of LangGraph's TypedDict with automatic reducers
  • Less opinionated — more design decisions land on the developer; patterns aren't enforced
  • Less observability — no native equivalent of LangSmith tracing yet
Use Cases

When to Use Agent Framework

Agent Framework shines when you need flexibility, simplicity, and multiple orchestration patterns without heavy framework overhead.

🔬 Research Pipelines

Dynamic routing between specialists maps naturally to the Handoff pattern. Call researcher → analyzer → writer → critic without a full state machine.

💬 Conversational Agents

Group Chat pattern for multi-agent debates, customer service routing, or collaborative brainstorming with natural speaker-selection logic.

🧠 Autonomous Planning

Magentic One for complex long-horizon tasks requiring a planner that decomposes work, delegates to specialists, and synthesizes results.

⚡ Parallel Processing

Concurrent pattern for independent subtasks — research multiple topics simultaneously, then merge results in a final synthesis agent.

🛡️ Production Guardrails

Middleware pipeline for rate limiting, content moderation, PII detection, and safety filters — plugged in per-agent without changing agent logic.

🎓 Teaching Agent Concepts

Simple API makes orchestration patterns visible without boilerplate. Students understand the handoff concept in minutes, not hours.

Getting Started

Quick Start

Install & Run

# Install dependencies
pip install agent-framework \
            agent-framework-openai

# Configure API key
cp .env.example .env
# → set OPENAI_API_KEY in .env

# Run the research system
python runner.py \
  --topic "quantum computing" \
  --mode basic

# Streaming mode
python runner.py \
  --topic "AI in healthcare" \
  --mode stream --depth deep

# Human-in-the-loop mode
python runner.py \
  --topic "climate change" \
  --mode hitl

Minimal Agent + Tool

from agent_framework.openai import (
    OpenAIChatClient
)
from agent_framework import tool
from typing import Annotated

@tool
def web_search(
    q: Annotated[str, "search query"]
) -> str:
    return "search results..."

@tool(approval_mode="always_require")
def publish(
    text: Annotated[str, "report text"]
) -> str:
    """Requires human approval."""
    return "Published!"

client = OpenAIChatClient(
    model="gpt-4o-mini",
    api_key="sk-..."
)
agent = client.as_agent(
    name="Researcher",
    instructions="You are a specialist.",
    tools=[web_search, publish],
)

# Non-streaming
result = await agent.run("Research AI")

# Streaming
async for chunk in agent.run(
    "Research AI", stream=True
):
    if chunk.text:
        print(chunk.text, end="")
