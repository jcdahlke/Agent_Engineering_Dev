Overview

What is LangGraph?

A low-level framework by LangChain for building stateful, cyclical multi-agent systems as explicit graphs.

🔄

Cycles & Loops

Unlike simple DAG frameworks, LangGraph supports cycles — agents can loop, retry, and revisit steps.

💾

Persistent State

Every node reads and writes a shared typed state. Checkpointing saves state at every step for resume and time-travel.

🎮

Human-in-the-Loop

First-class interrupt() support — pause mid-execution for human review, then resume seamlessly.

🌊

Streaming

Stream token-by-token LLM output, per-node state updates, or custom structured events in real time.

🗺️

Visualization

Built-in Mermaid and ASCII diagram generation. Visualize exactly what your graph does before running it.

⚡

Production-Ready

RetryPolicy, error handling, async support, SQLite persistence, and LangSmith tracing out of the box.

How It Works

Agent Roles

🎯 Supervisor
gpt-4o-mini · temperature 0
  • Reads full state, decides what to do next
  • Uses handoff tools — LLM picks call_researcher, call_writer, etc.
  • Tool call name becomes the routing signal
  • Saves/reads long-term memory via InMemoryStore
🔍 Researcher
gpt-4o-mini · manual ReAct loop
  • Tavily / DuckDuckGo web search
  • arXiv academic paper search
  • Wikipedia background lookup
  • Page scraping via BeautifulSoup
  • Emits live progress via StreamWriter
📊 Analyzer
gpt-4o · structured output (Pydantic)
  • Reads all raw content from state
  • Runs Python code via PythonREPLTool
  • Returns typed AnalysisResult with confidence score
  • Signals if more research is needed
✍️ Writer
gpt-4o · structured output (Pydantic)
  • Synthesizes findings → structured ResearchReport
  • Renders to Markdown with citations
  • Addresses critique from previous cycles
  • Incorporates human feedback when provided
🔍 Critic
gpt-4o-mini · structured output + interrupt()
  • Scores draft 0.0–1.0 with rubric
  • Returns typed CritiqueResult with strengths/weaknesses
  • Approves (≥0.7) → graph exits to END
  • Calls interrupt() when score <0.4 or sensitive topic

Shared State Design

Every node reads and writes ResearchState. Annotated reducers control how updates are merged.

FieldTypeReducerPurpose
messageslist[BaseMessage]add_messagesFull conversation; deduplicates by ID
sources_foundlist[str]operator.addURLs accumulated across all researcher calls
raw_contentlist[str]operator.addScraped text chunks, appended each run
key_findingslist[str]operator.addBullet findings, accumulated from analyzer
report_draftstroverwriteLatest draft — writer replaces each cycle
quality_scorefloatoverwriteMost recent critic score (0–1)
task_completebooloverwriteSupervisor sets True to trigger END
human_feedbackOptional[str]overwritePopulated via interrupt() resume value

operator.add

List fields accumulate — no node needs to read-then-append manually.

add_messages

Deduplicates by message ID — handles OpenAI tool-call round-trips correctly.

overwrite

Scalar fields like report_draft are the current state — last write wins.

Evaluation

Pros & Cons of LangGraph

✓ Strengths

  • Fine-grained control — you own every edge and routing decision; no magic
  • First-class HITL — interrupt() pauses mid-node, full state preserved
  • Persistence built in — MemorySaver/SqliteSaver checkpoint every step automatically
  • Multiple streaming modes — tokens, node updates, custom events
  • Visualizable — ASCII + Mermaid diagrams generated from the graph itself
  • Time-travel debugging — inspect or rewind to any past checkpoint
  • Production features — RetryPolicy, async, LangSmith tracing
  • Framework-agnostic nodes — any Python function can be a node

✗ Limitations

  • Verbose boilerplate — simple tasks need a lot of scaffolding vs. CrewAI/AutoGen
  • Steep learning curve — reducers, state channels, and routing require deep understanding
  • No built-in planner — you must design the graph topology yourself
  • Overkill for simple tasks — a single LLM call doesn't need a graph
  • LangChain dependency — tight coupling to the LangChain ecosystem
  • Debugging complexity — tracing issues across many nodes can be difficult without LangSmith
  • Limited auto-parallelism — concurrent nodes require explicit Send fan-out
Use Cases

When to Use LangGraph

LangGraph shines when your agent needs cycles, state, and oversight.

🔬 Research & Analysis Pipelines

Multi-step information gathering, analysis, synthesis, and quality review — exactly this demo. Checkpointing lets long-running sessions survive restarts.

💻 Coding Assistants

Plan → write code → run tests → debug → rewrite loops. The cycle naturally maps to a LangGraph with conditional edges on test pass/fail.

🏢 Business Process Automation

Multi-step workflows where human approval gates must be embedded at specific points — contract review, compliance checks, expense approvals.

🤖 Autonomous Agents

Long-horizon tasks where an agent must plan, act, observe, and adapt over many steps. Persistent state across sessions is essential.

🧪 Evaluation Frameworks

Generate → evaluate → regenerate loops for LLM output quality control. The critic pattern generalizes to any self-improving system.

🎓 Teaching Agent Concepts

The explicit graph makes agent architectures visible and debuggable — ideal for education. Students see exactly how orchestration works.

Choose LangGraph when you need fine control, persistence, HITL, or streaming.   Choose CrewAI/AutoGen for faster prototyping with less boilerplate.
